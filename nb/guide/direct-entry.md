# Direktepåmelding

Time4o sin direktepåmelding kan brukes som eneste påmeldingsløsning,
eller som et suplement til f.eks. Eventor sin påmeldingsløsning.

Fordelen med Time4o sin påmeldingsløsning i motsetning til Eventor, er at den holder styr på antall ledige kart og antall ledige starttider.
Når Eventor har stengt sin påmelding og startlister er trukket kan direktepåmeldingen åpne for å ta i mot påmeldinger fram til det er tomt for kart eller starttider uten at arrangør må følge med.

Den er tilpasset mobilskjerm og arrangør kan skrive ut og henge opp en QR kode som peker til direktepåmeldingen.

Det er også mulig å selv endre brikkenummer i direktepåmeldingsløsningen, enten basert på [tilsendt e-post](/nb/guide/tutorials/entry-email) eller via en påmeldingsliste.

## Åpne direktepåmelding

1. Gå til ***Arrangement*** > ***Løp***
2. Åpne aktuelt løp og gå til fanen ***Direktepåmelding***.
3. Huk av for ***Tillat direktepåmelding***

- Link til direktepåmeldingen ligger på dashbordet.
- QR-kode kan lastes ned som PDF via link nederst på direktepåmeldingssiden.

Dette er alt som trengs for å åpne direktepåmeldingen. Det finnes derimot flere konfigueringsmuligheter. 


## E-postbekreftelse

Ved påmelding må deltageren oppgi e-postadressen. En påmeldingsbekreftelse med en link vil bli sendt til denne e-postadressen.
På løpet finnes det et felt man kan legge inn egen info som vises i e-posten.

Det er også mulig å sende ut påmeldingsbekreftelse til alle deltagere via følgende [oppgave](/nb/tasks/send-entry-email).

## Endringer

Hvis det åpnet for å tillate endringer, kan deltagerne endre påmeldingen sin via linken de mottar i e-postbekreftelsen.

Hvis påmeldingslisten publiseres kan deltagerne også endre sin påmelding via denne. Dette vil være aktuelt for deltagere som er påmeldt på andre måter, f.eks. via Eventor, og dermed ikke nødvendigvis har fått tilsendt en påmeldingsbekreftelse.

Merk at alle med linken til påmeldingslisten kan endre påmeldingen til alle deltagere.

::: warning OBS!
Foreløpig er det kun brikkenummer som kan endres. Det vil komme mulighet for å gjøre flere endringer. 
:::


## Klasser

- Deltageren kan kun melde seg på i klasser som deltageren er kvalifisert for basert på kjønn og alder, samt har ledige plasser.
- På løpet kan man velge om direktepåmeldingen kun skal være tilgjengelig for klasser med f.eks. fristart.
- På klassenivå kan man unnta klasser fra å ikke ha direktepåmelding (***Klasser*** > ***Direktepåmelding***)

## Påmeldingsbegrensning

Det er flere måter å begrense antall påmeldte i en klasse via direktepåmeldingen. Innstillingen ***Påmeldingsbegrensning*** kan settes på løpet og overstyres per klasse.

Når begrensningen er nådd vil det ikke være mulig å melde seg på i klassen.

| Påmeldingsbegrensning | Beskrivelse |
| :--- | :--- |
| **Ingen begrensning** | Klassen tar i mot ubegrenset antall påmeldinger. |
| **Ledige starter** | Klassen kan ta i mot påmeldinger så lenge det er ledige [starter](/nb/resources/event/starts). |
| **Ledige kart** | Klassen kan ta i mot påmeldinger så lenge det er ledige kart. Antall ledige kart oppgis på [løypene](/nb/resources/event/courses_controls). |
| **Maks antall påmeldinger (klasse)** | Antall påmeldinger totalt sett kan angis på klassen. Direktepåmeldingen stenges for klassen når maks antall påmeldinger er nådd. |
| **Kombinasjon** | Dette er den anbefalte metoden i de fleste sammenhenger. Når denne er valgt er det laveste verdi av antall ledige starter, ledige kart og maks antall påmeldinger som benyttes. Den hensyntar kun de alternativene som har satt en verdi. Dvs. om klassen ikke har noen starter, og det ikke er satt maks antall på klassen, så er det antall ledige kart som benyttes. Hvis heller ikke antall kart er satt, dvs. ingen verdier er satt, så tar klassen i mot ubegrenset antall påmeldinger. |


## Starttildeling

Direktepåmeldingen kan automatisk tildele ledige starter til påmeldte, eller den påmeldte kan selv velge starttid.

Innstillingen ***Starttildeling*** kan settes på løpet og overstyres per klasse.
